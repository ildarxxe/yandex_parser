# ReviewHub

Тестовое SPA на Laravel 13 и Vue 3 для подключения карточки организации в
Яндекс.Картах, фонового сбора отзывов и просмотра рейтинга со счётчиками.

## Возможности

- SPA-аутентификация через Laravel Sanctum;
- сохранение и валидация ссылки на карточку организации;
- фоновая синхронизация через database queue;
- сбор до 600 доступных отзывов;
- отдельные значения среднего рейтинга, количества оценок и отзывов;
- локальное хранение результата и API-пагинация по 50 отзывов;
- статусы загрузки и понятные ошибки в интерфейсе.

## Локальный запуск

Нужны Docker и Docker Compose. Остальные зависимости устанавливаются внутри
контейнеров.

```bash
docker compose up --build
```

Первый запуск создаёт `.env`, генерирует `APP_KEY`, устанавливает зависимости,
применяет миграции и запускает сиды. После готовности контейнеров приложение
доступно по адресу <http://localhost:8080>.

Тестовый пользователь: **admin / 123456**.

```text
login: admin
password: 123456
```

Остановка контейнеров:

```bash
docker compose down
```

Для полного удаления локальной базы и dependency volumes:

```bash
docker compose down -v
```

## Проверки

Backend-тесты используют SQLite in-memory и не обращаются к Яндексу:

```bash
docker compose run --rm --no-deps --entrypoint vendor/bin/phpunit app
```

Проверка стиля PHP:

```bash
docker compose run --rm --no-deps --entrypoint vendor/bin/pint app --test
```

Production-сборка frontend:

```bash
docker compose run --rm --no-deps frontend sh -c "npm ci && npm run build"
```

## Подход к парсингу

Headless-браузер для прототипа не используется. Публичная страница карточки уже
содержит сериализованное состояние в `script.state-view`; парсер извлекает его
через DOM, находит данные организации и повторяет внутреннюю пагинацию страницы
отзывов с параметром `page`.

Перед запросом ссылка нормализуется до `/reviews/`. Поддерживаются обычные и
короткие ссылки региональных доменов Яндекса. Поддерживается и share-формат,
в котором организация передаётся через `poi[uri]`:

```text
https://yandex.kz/maps/10295/kostanai/?mode=poi&poi%5Buri%5D=ymapsbm1%3A%2F%2Forg%3Foid%3D162508758578
```

Обычный формат карточки также подходит:

```text
https://yandex.ru/maps/org/yandeks/1124715036/reviews/
```

Для короткой ссылки разрешены только редиректы на домены Яндекса, чтобы не
превратить backend в произвольный HTTP-клиент.

Парсер последовательно загружает максимум 12 страниц по 50 отзывов, удаляет
дубликаты по внешнему ID и сохраняет:

- название и внешний ID организации;
- средний рейтинг;
- количество оценок и количество отзывов отдельно;
- автора, дату, текст и оценку каждого отзыва.

Ограничение в 12 страниц соответствует практическому пределу около 600 отзывов
из задания. Таймауты, повторы и предел страниц вынесены в переменные окружения.
Изменение структуры страницы, пустой ответ, неполные отзывы и сетевые ошибки
возвращаются как осмысленные ошибки синхронизации.

## Кэширование и очередь

Запрос пользователя только сохраняет ссылку и ставит `SyncOrganization` в
очередь. Отдельный worker получает все доступные страницы и атомарно заменяет
локальный набор отзывов в транзакции.

Frontend опрашивает статус организации, а после завершения читает отзывы из
PostgreSQL через `/api/reviews?page=N`. Внешний источник поэтому не вызывается
при каждом переключении страницы. Старый результат сохраняется, если источник
временно недоступен, а устаревшая job не может перезаписать данные для новой
ссылки.

## Структура

```text
app/Services/Yandex/YandexMapsDataSource.php  работа с Яндекс.Картами
app/Services/OrganizationSyncService.php      транзакционное сохранение
app/Jobs/SyncOrganization.php                 фоновая задача
app/Http/Controllers/Api                      JSON API
resources/js                                  Vue 3 Composition API
```

Основные таблицы: `users`, `organizations`, `reviews`, а также стандартные
таблицы Laravel для очереди, сессий и кэша.

## Переменные окружения

Полный пример находится в `.env.example`. Основные настройки:

| Переменная | Назначение | По умолчанию |
| --- | --- | --- |
| `APP_URL` | публичный URL приложения | `http://localhost:8080` |
| `DB_*` | подключение к PostgreSQL | контейнер `postgres` |
| `QUEUE_CONNECTION` | драйвер очереди | `database` |
| `YANDEX_MAPS_MAX_PAGES` | число страниц парсинга, не более 12 | `12` |
| `YANDEX_MAPS_TIMEOUT` | общий timeout HTTP-запроса, секунды | `20` |
| `YANDEX_MAPS_CONNECT_TIMEOUT` | timeout подключения, секунды | `5` |
| `YANDEX_MAPS_RETRY_TIMES` | число повторов HTTP-запроса | `2` |
| `YANDEX_MAPS_RETRY_DELAY` | пауза между повторами, мс | `500` |

## Развёртывание

`compose.yaml` ориентирован на локальную разработку и поднимает Vite dev server.
Для production нужно собрать frontend командой `npm ci && npm run build`,
выставить `APP_ENV=production`, `APP_DEBUG=false`, публичный `APP_URL` и
production-доступы к PostgreSQL. Queue worker должен работать отдельным
долгоживущим процессом:

```bash
php artisan queue:work database --queue=sync --tries=1 --timeout=600
```

Перед запуском приложения выполняются:

```bash
composer install --no-dev --prefer-dist --optimize-autoloader
php artisan migrate --force
php artisan optimize
```
